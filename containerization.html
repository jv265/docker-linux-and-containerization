What is it?

Containerization, a light-weight process that allows applications to be "written once and run everywhere" in simple terms. An alternative to virtualization, allows packaging of software and any dependencies into containers of sorts that is abstracted away from the OS. It's able to run cross platform and across any suitable means of the cloud, virtual or physical machines.

Creds: https://hackernoon.com/hn-images/1*VxAJ5aCzUJtmVhdg290RXg.png

With containers, CPU, memory, storage and network resources at the OS level are virtualized to provide developers a sandboxed view logically separated from other applications. Any number of applications can be run through X many containers which are all managed through a container engined hosted through a Host OS.

Where does docker come in?

Docker is a containerization platform that allows one to be able to create, deploy and run applications using containers. It packages one's application and all of the associated dependencies together in a container(s) to work seamlessly in any environment. Mainly developed for Linux, uses the resource isolation features of the Linux Kernel. It allows for complete monitor and management of containers using the Docker API and uses low CPU/Memory overhead.
